# OFFICE_HANDOFF

## 用途說明
這是一套固定在本機使用的 Demo 版「飯店合作夥伴入場管理系統」。

目前做法：
- 系統固定開在這一台電腦
- 資料保存在這台電腦的瀏覽器 `localStorage`
- 同事直接使用這台電腦操作
- 目前唯一正式使用網址為 `http://localhost:5179/`

## 每日啟動 SOP

| 步驟 | 操作 |
| --- | --- |
| 1 | 打開 `PowerShell` |
| 2 | 輸入 `cd D:\hotel-partner-entry-system\web` |
| 3 | 輸入 `npm run dev` |
| 4 | 等終端機顯示網址後，用瀏覽器打開 |
| 5 | 一律使用 `http://localhost:5179/` |

## 如果網站已經開著
1. 直接打開瀏覽器
2. 輸入：
   `http://localhost:5179/`
3. 開始操作

## 不幸關機後的重新開啟方式

### 情境 A：電腦重開機了，網站打不開
請重新做一次：

1. 打開 `PowerShell`
2. 輸入：
   `cd D:\hotel-partner-entry-system\web`
3. 再輸入：
   `npm run dev`
4. 用瀏覽器打開：
   `http://localhost:5179/`

### 情境 B：網站頁面開著，但突然無法連線
請先確認：

1. `PowerShell` 視窗是不是被關掉了
2. 電腦是不是剛重開機
3. 現在開的是不是 `http://localhost:5179/`

如果不確定，最簡單做法就是：

1. 關掉舊分頁
2. 重新打開 `PowerShell`
3. 重新輸入：
   `cd D:\hotel-partner-entry-system\web`
4. 再輸入：
   `npm run dev`
5. 改用新的 `http://localhost:5179/` 網址

## 正確使用規則

| 情境 | 正確做法 |
| --- | --- |
| 要保留資料 | 只用 `5179` |
| 同事要操作 | 直接用這台電腦 |
| 要保留資料備份 | 依照 `EXPORT_DATA.md` 匯出 JSON |
| QR 掃不出來 | 改用序號手動搜尋 |

## Demo 操作順序

1. 到 `發放憑證`
2. 建立新的 QR
3. 下載 QR 圖檔
4. 到 `現場核銷`
5. 掃碼或用序號手動查找
6. 輸入本次核銷人數
7. 到 `管理後台` 查看統計變化

## 常見問題

### 為什麼資料會保留？
因為資料存在這台電腦的瀏覽器裡，只要不清除瀏覽器資料，就會保留。

### 如果同事要接手怎麼做？
直接來這台電腦操作，不需要另外安裝或複製專案。

### 如果 QR 掃不出來怎麼辦？
請先改用 QR 圖上的序號，在 `現場核銷` 頁手動搜尋對應憑證。

## 重要提醒
- 正式操作請只用 `5179`
- 不要清除瀏覽器資料
- 不要換瀏覽器
- 關機後網站不會自動重開，需要重新執行 `npm run dev`
